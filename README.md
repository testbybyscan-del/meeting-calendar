# Календарь встреч на PHP + PostgreSQL

**Управление встречами с авторизацией и календарём**

Версия: **1.0** (продакшен-готовность)

Проект представляет собой полнофункциональное веб-приложение для планирования встреч. Разработан на чистом PHP с использованием PostgreSQL и Docker. Позволяет создавать, редактировать, удалять встречи, просматривать их по датам. Система включает авторизацию пользователей и ведение аудита действий.

---

## Возможности

- Регистрация и вход (по умолчанию `admin` / `admin`)
- Интерактивный календарь с возможностью выбора даты
- Добавление встречи с указанием времени, адреса, описания
- Редактирование и удаление существующих встреч
- Фильтрация встреч по выбранной дате
- Логирование всех действий (добавление, изменение, удаление, ошибки)
- Полная контейнеризация для быстрого развёртывания
- Все учётные данные вынесены в `.env` для удобства настройки
- Работает в изолированной сети Docker

---

## Технологии

- **PHP 8.2** (чистый код, без фреймворков)
- **PostgreSQL 15** (вместо файлового хранилища)
- **Bootstrap 5** (адаптивный интерфейс)
- **Flatpickr** (удобный выбор дат)
- **Docker & docker-compose** (контейнеризация)
- **PDO** (безопасные подготовленные запросы)

## Быстрый старт (Docker)

1. Установите **Docker** и **docker-compose**.
2. Склонируйте репозиторий или создайте папку с файлами проекта.
3. Скопируйте `.env.example` в `.env` и при необходимости отредактируйте параметры (пароли, порты и т.п.):
   ```bash
   cp .env.example .env
```

4. В терминале из корневой папки выполните:
   ```bash
   docker-compose up -d
   ```
5. Откройте браузер по адресу:
   - **`http://localhost:8080`** (если порт 8080 свободен)
   - Если порт 8080 занят – измените в `docker-compose.yml` на любой свободный (например, 8081) и перезапустите контейнеры.
6. Войдите с учётными данными:
   - **Логин:** `admin`
   - **Пароль:** `admin`

> **Примечание:** если вы видите страницу «It works!» вместо приложения, убедитесь, что у Apache включён `DirectoryIndex` (это уже настроено в образе) или проверьте, что порт действительно принадлежит Docker-контейнеру (выполните `docker ps`).

## Настройка вручную (без Docker)

Если вы хотите запустить проект без Docker:

1. Установите PHP 8.2+ и PostgreSQL.
2. Создайте базу данных и выполните скрипт `init.sql`.
3. Скопируйте `.env.example` в `.env` и укажите свои параметры подключения.
4. Поместите все файлы проекта в веб-директорию (например, `/var/www/html`).
5. Убедитесь, что веб-сервер (Apache/Nginx) имеет доступ к чтению файлов и настроен на обработку PHP.
6. Откройте `index.php` в браузере.

## Переменные окружения (.env)

Файл `.env` содержит следующие настройки:

```ini
# PostgreSQL
POSTGRES_DB=postgres
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres

# Для PHP-приложения (те же значения, что и для БД)
DB_HOST=db
DB_PORT=5432
DB_NAME=${POSTGRES_DB}
DB_USER=${POSTGRES_USER}
DB_PASSWORD=${POSTGRES_PASSWORD}
```

Все параметры используются как в `docker-compose.yml`, так и в `config.php` через `getenv()`.
При необходимости вы можете изменить порт для веб-сервиса, добавив в `docker-compose.yml` строчку `ports: - "НовыйПорт:80"`.

## Описание работы

- **Авторизация** – проверка логина/пароля через `password_verify()` (поддерживается и MD5 для обратной совместимости).
- **Встречи** – хранятся в таблице `meetings` с полями: дата, время, адрес, описание, создатель, дата изменения и т.д.
- **Календарь** – генерируется на сервере, клиентский flatpickr используется для навигации.
- **Действия** – все операции (добавление, редактирование, удаление, ошибки) записываются в таблицу `actions_log` с указанием времени, IP, пользователя и деталей.
- **Безопасность** – все SQL-запросы выполняются через подготовленные выражения (PDO), что предотвращает SQL-инъекции.

## Устранение возможных проблем

| Проблема | Решение |
|----------|---------|
| Страница «It works!» вместо календаря | Убедитесь, что `DirectoryIndex index.php` включён (в образе это уже настроено). Проверьте, что вы открываете правильный порт. |
| Ошибка подключения к БД | Проверьте переменные в `.env`, особенно `DB_HOST=db` (для Docker) и пароли. Перезапустите контейнеры. |
| Неверные учётные данные при входе | Сбросьте пароль в БД: `UPDATE users SET password = MD5('admin') WHERE username = 'admin';` (для теста). |
| Конфликт порта 8080 | Измените порт в `docker-compose.yml` на 8081 или любой другой свободный. |
| Ошибка `Cannot redeclare logAction()` | Убедитесь, что функция `logAction()` объявлена только в `config.php` (в `auth.php` она удалена). |
| Контейнер `db` перезапускается | Используйте образ `postgres:15` (не `latest`), удалите старый том и пересоздайте. |
